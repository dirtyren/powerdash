# P3.3 — PromQL for non-series kinds (KPI / Gauge / Table / Pie / Donut) Design

**Status:** Design approved 2026-05-01 — ready for implementation plan.
**Depends on:** P3.1 (Prometheus proxy routes, schemas), P3.2 (WidgetRef.query, QueryEditor, render dispatcher).
**Estimate:** ~13.5 engineering hours (~1.75 days).

Phase 3 of 3 for the Prometheus integration. P3.1 shipped the proxy + demo, P3.2 wired 5 series kinds and a per-widget editor, and P3.3 extends the dispatcher to 5 non-series kinds that take instant-vector responses.

---

## 1. Goal

Let a user attach a PromQL expression to a KPI / Gauge / Table / Pie / Donut widget and see live Prometheus data. Query editor UX is unchanged from P3.2 — same textarea, same Apply button. The dispatcher learns five more renderers, each consuming the existing `POST /api/promql/query` proxy (instant vector).

## 2. Scope

**In scope**
- Wire 5 non-series kinds: KPI, Gauge, Table, Pie, Donut.
- New `useQueryInstant` hook mirroring P3.2's `useQueryRange`, POSTing to `/api/promql/query`.
- Five new renderer components under `src/components/widgets/promql/`: `PromKpi`, `PromGauge`, `PromTable`, `PromPie`, `PromDonut`.
- Extract shared `labelFor` + `parseValue` helpers to `src/widgets/promql/instant-helpers.ts`; `series-option.ts` re-uses them.
- Extend `WidgetByKind` in `src/widgets/render.tsx` with a kind→instant-renderer map.
- E2E extension: attach a query to the KPI widget (`w-cpu-kpi`), Apply, assert a number renders.
- Unit tests where happy-dom can verify behavior (KPI text, Table columns, Pie/Donut option shape).

**Out of scope** (deferred)
- Reducer dropdown for KPI/Gauge. They display `result[0].value[1]`. Multi-series hint: "Showing 1 of N series" below the number. Users aggregate via PromQL.
- The remaining 10 non-series kinds: **radar, funnel, tree, sunburst, treemap, heatmap, sankey, histogram, box-plot, candlestick**. Silent fall-through to their sample rendering if a query is set.
- Per-kind label customization (e.g., choose which metric label becomes a pie slice name). Slice/series labels are Grafana-style, built from non-`__name__` labels.
- Configurable table columns. Columns are the union of label keys across results plus a `Value` column.
- Hiding `step` from the QueryEditor when the selected kind is non-series.
- Gauge thresholds / color zones.
- Number formatting controls (unit suffix, decimal places — KPI renders `.toFixed(1)`, Table `.toFixed(3)`, hard-coded).

## 4. `useQueryInstant` hook

```ts
// src/hooks/useQueryInstant.ts
"use client";

import { useQuery } from "@tanstack/react-query";
import type { PromInstantResponse } from "@/server/schemas/prometheus";

export interface QueryInstantOptions {
  time?: number;            // unix seconds
  refetchIntervalMs?: number;
}

async function fetchInstant(
  expr: string,
  opts: QueryInstantOptions,
): Promise<PromInstantResponse> {
  const r = await fetch("/api/promql/query", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    credentials: "include",
    body: JSON.stringify({ expr, time: opts.time }),
  });
  if (!r.ok) {
    const text = await r.text();
    throw new Error(`PromQL query failed: ${r.status} ${text}`);
  }
  return (await r.json()) as PromInstantResponse;
}

export function useQueryInstant(expr: string, opts: QueryInstantOptions = {}) {
  return useQuery({
    queryKey: ["promql-instant", expr, opts.time],
    queryFn: () => fetchInstant(expr, opts),
    refetchInterval: opts.refetchIntervalMs ?? 15_000,
    enabled: expr.trim().length > 0,
  });
}
```

Mirror of `useQueryRange` with the `/api/promql/query` endpoint and a simpler options surface (no `start/end/step`).

## 5. Shared helpers

Extract the existing helpers from `src/widgets/promql/series-option.ts` to a new module:

```ts
// src/widgets/promql/instant-helpers.ts

export function labelFor(metric: Record<string, string>): string {
  const { __name__, ...rest } = metric;
  const tail = Object.entries(rest)
    .map(([k, v]) => `${k}="${v}"`)
    .join(", ");
  if (__name__) return tail ? `${__name__}{${tail}}` : __name__;
  return tail || "series";
}

export function parseValue(v: string): number | null {
  const n = parseFloat(v);
  return Number.isFinite(n) ? n : null;
}
```

Then `series-option.ts` imports `{ labelFor, parseValue }` from `./instant-helpers`. Its existing tests still pass (behavior unchanged). No new test file for the helpers — the existing `series-option.test.ts` already exercises them; adding duplicate coverage here is noise.

## 6. Renderer components

### 6.1 `PromKpi.tsx`

```tsx
"use client";

import type { WidgetRef } from "@/server/schemas/widget";
import { useQueryInstant } from "@/hooks/useQueryInstant";
import { parseValue } from "@/widgets/promql/instant-helpers";

export function PromKpi({ widget }: { widget: WidgetRef }) {
  const expr = widget.query?.expr ?? "";
  const { data, isLoading, error } = useQueryInstant(expr);

  if (!expr) {
    return (
      <div className="text-muted-foreground text-sm">
        No query — select this widget and add a PromQL expression.
      </div>
    );
  }
  if (isLoading) return <div className="text-muted-foreground">loading…</div>;
  if (error) {
    return <div className="text-red-400 text-sm">Query failed: {error.message}</div>;
  }
  const results = data?.data.result ?? [];
  if (results.length === 0) {
    return <div className="text-muted-foreground text-sm">No samples.</div>;
  }
  const first = results[0]!;
  const parsed = parseValue(first.value[1]);
  return (
    <div className="flex flex-col items-baseline gap-1">
      <span className="text-3xl font-semibold">
        {parsed === null ? "—" : parsed.toFixed(1)}
      </span>
      {results.length > 1 && (
        <span className="text-muted-foreground text-xs">
          Showing 1 of {results.length} series
        </span>
      )}
    </div>
  );
}
```

### 6.2 `PromGauge.tsx`

```tsx
"use client";

import type { WidgetRef } from "@/server/schemas/widget";
import { useQueryInstant } from "@/hooks/useQueryInstant";
import { parseValue, labelFor } from "@/widgets/promql/instant-helpers";
import { EchartsWidget } from "@/components/widgets/EchartsWidget";
// gauge series module is already registered in src/widgets/adapters/gauge.ts
// (via `echarts.use([GaugeChart])`) — importing the gauge adapter file here
// via the widgets registry (already loaded for the default renderer) means
// we don't need a redundant echarts.use call at this component's module.

export function PromGauge({ widget }: { widget: WidgetRef }) {
  const expr = widget.query?.expr ?? "";
  const { data, isLoading, error } = useQueryInstant(expr);

  if (!expr) return <EmptyQuery />;
  if (isLoading) return <Loading />;
  if (error) return <Error message={error.message} />;
  const results = data?.data.result ?? [];
  if (results.length === 0) return <Empty />;

  const first = results[0]!;
  const parsed = parseValue(first.value[1]);
  return (
    <EchartsWidget
      option={{
        series: [{
          type: "gauge",
          progress: { show: true },
          detail: { valueAnimation: true, formatter: "{value}" },
          data: [{ value: parsed ?? 0, name: labelFor(first.metric) }],
        }],
      }}
    />
  );
}

// small helpers inlined for brevity; actual file can extract to a shared
// `PlaceholderStates.tsx` if the repetition becomes painful across the 5 files.
function EmptyQuery() { return <div className="text-muted-foreground text-sm">No query — select this widget and add a PromQL expression.</div>; }
function Loading() { return <div className="text-muted-foreground">loading…</div>; }
function Error({ message }: { message: string }) { return <div className="text-red-400 text-sm">Query failed: {message}</div>; }
function Empty() { return <div className="text-muted-foreground text-sm">No samples.</div>; }
```

The 4 placeholder sub-components are duplicated across renderers. If the repetition becomes painful, the implementer can extract them — but optional for the MVP.

### 6.3 `PromTable.tsx`

```tsx
"use client";

import { useMemo } from "react";
import { type ColumnDef, flexRender, getCoreRowModel, useReactTable } from "@tanstack/react-table";
import type { WidgetRef } from "@/server/schemas/widget";
import { useQueryInstant } from "@/hooks/useQueryInstant";
import { parseValue } from "@/widgets/promql/instant-helpers";

type Row = Record<string, string>;

export function PromTable({ widget }: { widget: WidgetRef }) {
  const expr = widget.query?.expr ?? "";
  const { data, isLoading, error } = useQueryInstant(expr);

  const { columns, rows } = useMemo(() => {
    const results = data?.data.result ?? [];
    if (results.length === 0) return { columns: [] as ColumnDef<Row>[], rows: [] as Row[] };

    // Union of label keys across all results, excluding `__name__`.
    const labelKeys = new Set<string>();
    for (const r of results) {
      for (const k of Object.keys(r.metric)) {
        if (k !== "__name__") labelKeys.add(k);
      }
    }
    const sortedLabels = [...labelKeys].sort();

    const columns: ColumnDef<Row>[] = [
      ...sortedLabels.map((key) => ({
        accessorKey: key,
        header: key,
        cell: (info: { getValue: () => unknown }) => {
          const v = info.getValue();
          return <>{v == null ? "" : String(v)}</>;
        },
      })),
      {
        accessorKey: "__value",
        header: "Value",
        cell: (info: { getValue: () => unknown }) => {
          const v = info.getValue() as string;
          const parsed = parseValue(v);
          return <>{parsed === null ? "—" : parsed.toFixed(3)}</>;
        },
      },
    ];

    const rows: Row[] = results.map((r) => {
      const row: Row = {};
      for (const k of sortedLabels) row[k] = r.metric[k] ?? "";
      row.__value = r.value[1];
      return row;
    });

    return { columns, rows };
  }, [data]);

  const table = useReactTable({ data: rows, columns, getCoreRowModel: getCoreRowModel() });

  if (!expr) { /* placeholder */ return <div className="text-muted-foreground text-sm">No query — select this widget and add a PromQL expression.</div>; }
  if (isLoading) return <div className="text-muted-foreground">loading…</div>;
  if (error) return <div className="text-red-400 text-sm">Query failed: {error.message}</div>;
  if (rows.length === 0) return <div className="text-muted-foreground text-sm">No samples.</div>;

  return (
    <table className="w-full text-sm">
      <thead>
        {table.getHeaderGroups().map((hg) => (
          <tr key={hg.id} className="border-border text-muted-foreground border-b text-left">
            {hg.headers.map((h) => (
              <th key={h.id} className="px-2 py-2 font-medium">
                {flexRender(h.column.columnDef.header, h.getContext())}
              </th>
            ))}
          </tr>
        ))}
      </thead>
      <tbody>
        {table.getRowModel().rows.map((row) => (
          <tr key={row.id} className="border-border/50 border-b">
            {row.getVisibleCells().map((cell) => (
              <td key={cell.id} className="px-2 py-2">
                {flexRender(cell.column.columnDef.cell, cell.getContext())}
              </td>
            ))}
          </tr>
        ))}
      </tbody>
    </table>
  );
}
```

### 6.4 `PromPie.tsx` + `PromDonut.tsx`

Both render an echarts pie — Pie with `radius: "60%"`, Donut with `radius: ["40%", "70%"]`. Share a common builder:

```ts
// src/widgets/promql/pie-option.ts
import type { EChartsCoreOption } from "@/widgets/echarts-core";
import type { PromInstantResponse } from "@/server/schemas/prometheus";
import { labelFor, parseValue } from "./instant-helpers";

type Radius = string | [string, string];

export function buildPieOption(resp: PromInstantResponse, radius: Radius): EChartsCoreOption {
  const data = resp.data.result.map((r) => ({
    name: labelFor(r.metric),
    value: parseValue(r.value[1]) ?? 0,
  }));
  return {
    tooltip: { trigger: "item" },
    legend: { textStyle: { color: "#cbd5e1" } },
    series: [{ type: "pie", radius, data }],
  };
}
```

Then:
```tsx
// src/components/widgets/promql/PromPie.tsx
export function PromPie({ widget }: { widget: WidgetRef }) {
  return <PieLike widget={widget} radius="60%" />;
}

// src/components/widgets/promql/PromDonut.tsx
export function PromDonut({ widget }: { widget: WidgetRef }) {
  return <PieLike widget={widget} radius={["40%", "70%"]} />;
}
```

Where `PieLike` is a small helper component in `PromPie.tsx` (or colocated in a shared file) that runs the hook, handles the four placeholder states, and renders `<EchartsWidget option={buildPieOption(data, radius)} />`.

## 7. Dispatcher wiring

```tsx
// src/widgets/render.tsx  (modified)
import type { ComponentType } from "react";
import type { WidgetRef, WidgetKind } from "@/server/schemas/widget";
import { WIDGET_ADAPTERS } from "@/widgets/adapter";
import { EchartsWidget } from "@/components/widgets/EchartsWidget";
import { SeriesChart } from "@/components/widgets/SeriesChart";
import { PromKpi } from "@/components/widgets/promql/PromKpi";
import { PromGauge } from "@/components/widgets/promql/PromGauge";
import { PromTable } from "@/components/widgets/promql/PromTable";
import { PromPie } from "@/components/widgets/promql/PromPie";
import { PromDonut } from "@/components/widgets/promql/PromDonut";

const INSTANT_RENDERERS: Partial<
  Record<WidgetKind, ComponentType<{ widget: WidgetRef }>>
> = {
  kpi: PromKpi,
  gauge: PromGauge,
  table: PromTable,
  pie: PromPie,
  donut: PromDonut,
};

export function WidgetByKind({ widget }: { widget: WidgetRef }) {
  const adapter = WIDGET_ADAPTERS[widget.kind];
  if (widget.query?.expr) {
    if (adapter.seriesType) {
      return <SeriesChart widget={widget} seriesType={adapter.seriesType} />;
    }
    const Instant = INSTANT_RENDERERS[widget.kind];
    if (Instant) return <Instant widget={widget} />;
  }
  if (adapter.Renderer) return <adapter.Renderer widget={widget} />;
  if (adapter.buildOption) return <EchartsWidget option={adapter.buildOption(widget)} />;
  return <div className="text-red-400">Unsupported: {widget.kind}</div>;
}
```

Kinds not in `INSTANT_RENDERERS` and without `seriesType` fall through to their existing sample/legacy rendering even when a query is set. Documented in spec §8 risks.

## 10. Risks

1. **Unused `step` field in QueryEditor for non-series kinds.** A user setting `step` on a KPI is confused when nothing happens. Mitigation: the field remains visible but inert; hiding it per-kind is deferred polish.
2. **Silent fall-through for 10 unsupported non-series kinds.** If a user attaches a query to a radar/funnel/heatmap/etc., the query is stored but ignored at render time. No UI hint in MVP. Acceptable for a demo; document in user-facing notes later.
3. **Pie/Donut with many series** — Prometheus queries like `up` can return dozens of instance slices. The pie legend overflows, the chart becomes illegible. Users aggregate in PromQL (e.g., `sum by (job) (up)`) to reduce slice count. No client-side limit in MVP.
4. **Table column explosion** — same risk as Pie. Dozens of labels = dozens of narrow columns. Users use `label_replace` or label filters in PromQL. No column pinning / horizontal scroll affordance yet.
5. **Duplicated placeholder sub-components across 5 files.** `EmptyQuery / Loading / Error / Empty` repeat. Trade-off: extracting to a shared file costs a small amount of coupling. Optional cleanup; not blocking.
6. **`useQueryInstant` duplicates `useQueryRange`'s boilerplate.** A generic factory would DRY them. Flat files are easier to read at 20 lines each; accept duplication.
7. **Pie slice values of 0 still render with legends.** A result with a NaN value maps to 0 and appears in the legend. Honest about the data but visually noisy. Filtering is deferred.
8. **No number formatting controls.** KPI is `.toFixed(1)`, Table is `.toFixed(3)`. A "bytes" metric displays `12345678.0` instead of `12.3 MB`. Users can use PromQL's math (`/ 1024 / 1024`) to shape the value. Unit formatting is a Grafana feature; deferred.

## 11. Open questions (deferred)

- **Reducer UI for KPI/Gauge.** Taking the first result is the MVP; Grafana-style dropdown ("last/sum/mean/max/min") is a possible P3.4.
- **Gauge thresholds** (red below X, green above Y). Visual polish, deferred.
- **Step-field visibility per-kind.** Conditional rendering based on adapter type is small but unrequested for P3.3.
- **Hints for unsupported kinds with queries** (radar, funnel, etc.). User-facing affordance deferred.
- **Number formatters** (unit, decimals) on KPI / Gauge / Table. Grafana feature, deferred.
