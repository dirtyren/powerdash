# P2.4 Dashboard Rename (in edit mode) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let a user rename an existing dashboard by editing its name inline in the existing edit-mode toolbar; Save commits name + widget layout in a single PUT.

**Architecture:** Extend `PutBodySchema` on the `PUT /api/dashboards/[id]` route to accept an optional `name` field and forward it into `saveDashboard`. Widen `useSaveDashboard.mutate` from `(widgets)` to `({widgets, name?})`. The edit page tracks an `editName` state initialized from `data.name`, expands `isDirty` to flip on name changes, and passes `onTitleChange={setEditName}` to `<EditToolbar>` (that prop already exists — it was added in P2.2 but wasn't wired on the edit page).

**Tech Stack:** Next.js 15 App Router, TypeScript 5.6, Zod 3, TanStack Query 5, Vitest 2, Playwright 1.

**Spec:** `docs/superpowers/specs/2026-04-29-plan-2.4-dashboard-rename-design.md`

## Task 2: Widen `useSaveDashboard` + wire rename in the edit page

**Files:**
- Modify: `src/hooks/useSaveDashboard.ts`
- Modify: `app/dashboards/[id]/edit/page.tsx`

These are coupled: the hook signature change breaks its one caller (the edit page), and the edit page's new name state requires the hook to accept an object. Doing both in one commit keeps the tree green.

- [ ] **Step 1: Widen the hook**

Replace the entire contents of `src/hooks/useSaveDashboard.ts` with:

```ts
"use client";

import { useMutation, useQueryClient } from "@tanstack/react-query";
import type { Dashboard } from "@/server/schemas/dashboard";
import type { WidgetRef } from "@/server/schemas/widget";

export interface SaveDashboardInput {
  widgets: WidgetRef[];
  name?: string;
}

async function saveDashboardApi(id: string, input: SaveDashboardInput): Promise<Dashboard> {
  const r = await fetch(`/api/dashboards/${encodeURIComponent(id)}`, {
    method: "PUT",
    headers: { "Content-Type": "application/json" },
    credentials: "include",
    body: JSON.stringify(input),
  });
  if (!r.ok) {
    const text = await r.text();
    throw new Error(`save failed: ${r.status} ${text}`);
  }
  const data: Dashboard = (await r.json()) as Dashboard;
  return data;
}

export function useSaveDashboard(id: string) {
  const qc = useQueryClient();
  return useMutation({
    mutationFn: (input: SaveDashboardInput) => saveDashboardApi(id, input),
    onSuccess: (saved) => {
      qc.setQueryData(["dashboard", id], saved);
      void qc.invalidateQueries({ queryKey: ["dashboards"] });
    },
  });
}
```
